## 🌟 نبذة عن المشروع
**TOP 10** هي تجربة ويب تفاعلية بالكامل تتيح للأصدقاء التنافس في غرف خاصة لتخمين إجابات مرتبطة بعالم كرة القدم (مثل: الهدافين التاريخيين، الفائزين بالكرة الذهبية، الأندية الأكثر تتويجاً). اللعبة مبنية بالكامل باستخدام تقنيات الويب الأساسية وتعتمد على **Firebase Realtime Database** لإدارة المزامنة الفورية بين اللاعبين بدون الحاجة لبرمجة خادم خلفي معقد (Backend-less).

> **💡 تنويه:** هذا المشروع جاهز تماماً للاستخدام والنشر (Production-Ready).

---

## 🖥️ دورة اللعب وشاشات الموقع (Game Loop)

تنقسم اللعبة إلى 5 شاشات تفاعلية متناسقة، تُدار وتُحدث تلقائياً لكل اللاعبين في نفس الوقت:

### 1. الشاشة الرئيسية (Home Screen)
* **دخول اللاعب:** حقل مخصص لإدخال اسم اللاعب (متوافق تماماً مع اللغة العربية).
* **إنشاء غرفة (Host):** يمكن للاعب إنشاء غرفة لعب جديدة ليكون هو المضيف. يولد النظام كود غرفة فريد ومكون من 6 أحرف/أرقام تلقائياً.
* **الانضمام لغرفة (Join):** إدخال كود الغرفة المكون من 6 رموز للدخول إلى غرفة حالية نشطة (تستوعب حتى 10 لاعبين).

### 2. غرفة الانتظار (Lobby Screen)
* **مشاركة الكود:** زر مخصص لنسخ كود الغرفة بضغطة واحدة لمشاركته مع الأصدقاء.
* **قائمة اللاعبين (Live Roster):** تحديث حي وفوري لقائمة اللاعبين المنضمين مع شارات ملونة توضح من هو مضيف الغرفة.
* **بدء اللعب:** يظهر زر "بدء اللعبة" للمضيف فقط، بينما يرى باقي اللاعبين حالة الانتظار.

### 3. شاشة اللعب والسباق (Game Screen)
* **لوحة الأسئلة:** تعرض السؤال المختار وتصنيفه ورقم الجولة.
* **مؤشر الوقت الدائري (Live Timer Ring):** عداد تنازلي متزامن بدقة بين جميع اللاعبين بناءً على وقت خادم Firebase. يتغير لون المؤشر للتحذير عند اقتراب النهاية.
* **لوحة الإجابات الـ 10 المغلقة:** عند تخمين أي لاعب لإجابة صحيحة، تنقلب البطاقة فوراً لدى جميع اللاعبين لتكشف عن الإجابة وقيمتها النقطية واسم اللاعب الذي اكتشفها.
* **حقل الإدخال الذكي (Autocomplete):** مزود بمساعد إكمال تلقائي يقترح أسماء اللاعبين أو الأندية مع وجود **مشتتات (Decoys)** لرفع الحماس وتجنب التخمين العشوائي.
* **لوحة النقاط المصغرة (Mini Scoreboard):** قائمة جانبية تُحدث فوراً لترتيب اللاعبين ونقاطهم الحالية.

### 4. شاشة نتائج الجولة (Round Results)
* **كشف الإجابات:** تظهر الإجابات الصحيحة المميزة باللون الأخضر، وتُكشف الإجابات الفائتة باللون الأحمر.
* **ترتيب الجولة والترتيب العام:** تعرض ترتيب اللاعبين في الجولة المنتهية والترتيب التراكمي العام.

### 5. الشاشة النهائية للتتويج (Final Podium Screen)
* **منصة التتويج (Retro Podium):** تعرض أصحاب المراكز الثلاثة الأولى بتمثيل حركي مميز.
* **تأثير الاحتفال (Confetti Effect):** إطلاق قصاصات الورق الملونة احتفالاً بالفائز.
* **إعادة اللعب:** إمكانية إعادة ضبط نقاط الغرفة لبدء لعبة جديدة بنفس اللاعبين.

## 🎨 الهوية البصرية (Retro Arcade Theme)
تم تصميم اللعبة بهوية بصرية مستوحاة من ألعاب الأركيد الكلاسيكية:
* **ثيم النيون المتوهج (Neon Glow):** ألوان متناغمة (أخضر نيوني، وردي، سيان) مع تأثيرات زجاجية (Glassmorphism).
* **شاشات الـ CRT:** محاكاة لخطوط المسح الضوئي للشاشات القديمة (CRT Scanlines Overlay).
* **تأثيرات حركية دقيقة:** اهتزاز الشاشة عند الخطأ في الإجابة، وانفجارات البيكسل.

## 🛡️ حماية اللعب النظيف وتطبيع النصوص
* **تخمين عربي ذكي (Arabic Normalization):** نظام يحلل المدخلات ويعالج الفروق الشائعة (توحيد الهمزات، التاء المربوطة والهاء، الياء المقصورة).
* **مراقبة حماية اللعب (Anti-Cheat):** نظام يكتشف خروج اللاعب من علامة التبويب للبحث عن الإجابات في جوجل، يمنح اللاعب 3 إنذارات قبل حظره من الإجابة.
* **نظام المضيف (Host Migration):** إذا انقطع اتصال المضيف، تنتقل الإدارة تلقائياً للاعب التالي لضمان عدم توقف اللعبة.

## 📊 لوحة التحكم والإدارة (Admin Dashboard)
تتوفر لوحة تحكم متكاملة للمشرفين على الرابط `/dashboard` لمراقبة الأداء:
* **إحصائيات تاريخية (Lifetime Stats):** إجمالي الغرف، اللاعبين، الإجابات المكتشفة، وتصنيفات الأسئلة الأكثر شيوعاً.
* **مراقبة النشاط اللحظي (Live Analytics):** الغرف النشطة، متوسط اللاعبين لكل غرفة، وتحديد الغرفة "الأكثر سخونة".
* **أدوات التحكم:** إمكانية طرد اللاعبين فورياً (Instant Kick)، إنهاء الجولات قسرياً، والحذف اليدوي للغرف.
* **التنظيف الذكي (Auto-Cleanup):** خوارزمية تعمل في الخلفية لحذف الغرف الخاملة أو الفارغة لتوفير مساحة قاعدة البيانات.

> **🔒 أمان لوحة التحكم:** يتم تأمين الدخول برمز مرور. يوصى بشدة بتغيير رمز المرور الافتراضي المعين في ملف `public/js/dashboard.js` قبل النشر على بيئة الإنتاج.

## 🚀 طريقة التشغيل والتهيئة (Setup & Deployment)

### 1. إعداد قاعدة البيانات (Firebase):
1. أنشئ مشروعاً مجانياً في [Firebase Console](https://console.firebase.google.com/).
2. فعّل خدمة **Realtime Database**.
3. قم بتعديل قواعد الأمان (Rules) مؤقتاً للتطوير:
   ```json
   {
     "rules": {
       ".read": true,
       ".write": true
     }
   }
   ```
4. احصل على بيانات الاتصال الخاصة بمشروعك (Config Object).

### 2. ربط المشروع وتشغيله محلياً:
1. قم بإنشاء ملف `firebase-config.js` داخل مجلد `public/js/` والصق فيه بيانات الاتصال الخاصة بك.
   > **⚠️ تحذير أمني هام:** لتجنب سرقة مواردك، تأكد من إضافة مسار ملف `public/js/firebase-config.js` إلى ملف `.gitignore` قبل دفع التغييرات (Push) إلى GitHub!

2. ثبّت الاعتمادات وشغل خادم التطوير:
   ```bash
   npm install
   npm run dev
   ```
3. افتح المتصفح على: `http://localhost:3000`
   للوصول للوحة التحكم: `http://localhost:3000/dashboard`
